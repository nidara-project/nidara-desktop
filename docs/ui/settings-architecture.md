# Arquitectura del Panel de Ajustes (Settings)\n\nEspecificaciones para crear un panel de configuración propio, integrado visualmente pero funcionalmente robusto.\n\n## 1. Arquitectura Basada en JSON\nPara evitar crear un widget diferente para cada opción, el panel debe generarse dinámicamente desde un esquema JSON:\n```json\n{\n  \"network\": {\n    \"type\": \"switch\",\n    \"label\": \"Wifi\",\n    \"icon\": \"network-wireless\",\n    \"action\": \"toggle_wifi\"\n  },\n  \"theme\": {\n    \"type\": \"select\",\n    \"options\": [\"Dark\", \"Light\", \"Auto\"],\n    \"command\": \"gsettings set org.gnome.desktop.interface color-scheme ...\"\n  }\n}\n```\n\n## 2. Persistencia de Datos\n- **Configuración de Hyprland**: No editar `hyprland.conf` directamente si es posible. Usar archivos separados (ej. `~/.config/hypr/theme.conf`) y sobrescribirlos, o usar `hyprctl keyword` para cambios temporales y guardar en archivo al salir.\n- **Estado de la UI**: Guardar preferencias de la interfaz (posición del dock, opacidad) en `~/.cache/crystal-shell/state.json`.\n\n## 3. Integración con Hardware\n- **Brillo/Volumen**: Usar los servicios nativos de Astal (`Wp` para Wireplumber, `Brightness` si existe o leer `/sys/class/backlight`).\n- **Bluetooth**: Usar `bluez` vía DBus.\n
